# LSTM-Based Adversarial Attacks and Robustness Analysis for Time-Series Forecasting

This repository is a course reproduction and extension project focused on adversarial attacks and robustness analysis for time-series forecasting.

The current codebase contains:

- a power-consumption forecasting branch based on stacked LSTM models
- a traffic forecasting branch based on both LSTM and BiLSTM models
- standard gradient-based attacks: `FGSM` and `BIM`
- key-time-step attacks: `KTSA-FGSM` and `KTSA-BIM`
- traffic-side analysis scripts for saliency visualization, top-ratio selection, and random-step comparison

## Scope

The project currently studies:

- regression-based time-series forecasting
- adversarial perturbations on sequential inputs
- attack effectiveness versus perturbation sparsity
- saliency-guided key time-step attacks

## Datasets

### Power dataset

The power branch reads from:

- `Dataset/Power conumption dataset/household_power_consumption.txt`
- or, if the text file is not present, `Dataset/Power conumption dataset/household_power_consumption.zip`

The current workspace contains the zip file version.

Current preprocessing behavior in code:

- source columns:
  - `Global_active_power`
  - `Global_reactive_power`
  - `Voltage`
  - `Global_intensity`
  - `Sub_metering_1`
  - `Sub_metering_2`
  - `Sub_metering_3`
- target: next-step `Global_active_power`
- resampling: hourly mean with `df.resample('h').mean()`
- supervised framing: `1` input time step with `7` features
- split in `utils.power_data()`:
  - first `365 * 72` rows for training
  - remaining rows for held-out data
- further split in `Power_LSTM.py`:
  - first `8/12` of the held-out portion becomes validation
  - remaining `4/12` becomes test

### Traffic dataset

The traffic branch tries these paths in order:

- `Dataset/PEMS08-Traffic-Flow-Forecasting-main/traffic_dataset.csv`
- fallback: `Dataset/PEMS08 traffic flow dataset/traffic_dataset.csv`

The current workspace contains the fallback path version:

- `Dataset/PEMS08 traffic flow dataset/traffic_dataset.csv`

Current preprocessing behavior in code:

- input features: `flow`, `occupy`, `speed`
- target: next-step `speed`
- lookback window: `12` time steps
- split:
  - `70%` train
  - `10%` validation
  - `20%` test

## Main Scripts

Run all scripts from the `src/` directory because most paths are written relative to that location.

```powershell
cd src
```

### Power branch

- `Power_LSTM.py`
  Trains the power-consumption LSTM model, saves `../Trained models/Power_regression_LSTM.h5`, and writes training plots to `../Images/`.
- `Power_LSTM_test.py`
  Loads the saved power LSTM model and evaluates Clean, `FGSM`, `BIM`, `KTSA-FGSM`, and `KTSA-BIM`.

### Traffic LSTM branch

- `Traffic_LSTM.py`
  Trains the traffic LSTM model, saves `../Trained models/Traffic_regression_LSTM.h5`, and writes training plots to `../Images/`.
- `Traffic_LSTM_test.py`
  Loads the saved traffic LSTM model and evaluates Clean, `FGSM`, `BIM`, `KTSA-FGSM`, and `KTSA-BIM`.
- `Traffic_Saliency_Vis.py`
  Visualizes gradient saliency for the first traffic test sample and highlights the top `10%` key time steps.
- `Traffic_LSTM_optimal_top_ratio_plot.py`
  Scans candidate `top_ratio` values `[0.05, 0.10, 0.20, 0.30, 0.50]`, selects the best ratio for `KTSA-FGSM` and `KTSA-BIM`, and saves a comparison figure.
- `Traffic_Random_Exp.py`
  Compares saliency-guided KTSA attacks against random time-step attacks across multiple `top_ratio` settings.

### Traffic BiLSTM branch

- `Traffic_BiLSTM.py`
  Trains the traffic BiLSTM model and saves `Trained models/Traffic_regression_BiLSTM.h5`.
- `Traffic_BiLSTM_Attack_Comparison.py`
  Loads the BiLSTM model and plots Clean, `FGSM`, `BIM`, `KTSA-FGSM`, and `KTSA-BIM` prediction curves on the traffic test set.
- `Traffic_BiLSTM_Saliency_Vis.py`
  Visualizes gradient saliency for the first traffic test sample under the BiLSTM model.
- `Traffic_BiLSTM_optimal_top_ratio_plot.py`
  Performs the same candidate-ratio scan as the LSTM version and saves the BiLSTM comparison figure.

## Code Provenance

This repository is not an original codebase written fully from scratch.

It is derived from multiple upstream sources:

1. The original repository accompanying the paper *Adversarial Examples in Deep Learning for Multivariate Time Series Regression*
   - <https://github.com/dependable-cps/adversarial-MTSR>
2. A community fork referenced during recovery and reorganization of missing implementation details
   - <https://github.com/forallx94/adversarial-MTSR>
3. A public repository containing processed PEMS08 traffic forecasting data
   - <https://github.com/JvThunder/PEMS08-Traffic-Flow-Forecasting>

This repository further cleans, adapts, and extends those materials for course use.

Please see [CODE_PROVENANCE.md](./CODE_PROVENANCE.md) for detailed notes.

## This Repository's Current Contributions

Compared with the upstream materials, the current repository adds or emphasizes:

- a runnable traffic forecasting branch based on single-station PEMS08 data
- a traffic `BiLSTM` baseline in addition to the LSTM baseline
- `KTSA-FGSM` and `KTSA-BIM` key-time-step attacks
- saliency visualization for attack interpretation
- candidate `top_ratio` scanning and automatic ratio selection
- random-step attack baselines for comparison
- cleaned local project packaging with `uv` and `pyproject.toml`
